# Contributing

## Workflow

1. Create a branch from main.
2. Implement changes with tests.
3. Run format, analyze, and test locally.
4. Open pull request with clear scope and screenshots for UI changes.

## Pull Request Checklist

- Issue linked or context provided
- Tests added/updated for behavior changes
- Golden updates included when UI changed intentionally
- Backward compatibility considered for persisted settings

## Code Style

- Prefer readable, composable widgets over deeply nested build methods.
- Keep business logic out of UI layer when possible.
- Add concise comments only for non-obvious logic.

## Ownership

Document module owners as your team grows.
